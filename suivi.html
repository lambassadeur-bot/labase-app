Bases de données par type de support culturel
Où l'app va chercher les infos selon le type d'œuvre et le mode (scan code-barres vs recherche par titre).
✅ fiable  ·  ⚠️ partiel / instable  ·  ❌ non disponible par cette méthode.
Type de supportScan code-barresRecherche titreBases interrogées
📖 Livre / Manga✅ ISBN✅Google Books + Open Library
💿 CD✅ EAN❌Discogs + MusicBrainz
🎵 Vinyle✅ EAN❌Discogs + MusicBrainz
🎬 Film / DVD❌✅OMDB
📺 Série❌✅OMDB
🎲 Jeu de société⚠️ UPCitemdb✅BoardGameGeek (titre)
🎮 Jeu vidéo⚠️ UPCitemdb⚠️ clé RAWG manquanteRAWG
📦 Autre / Collectible⚠️ UPCitemdb❌UPCitemdb (générique)
💡 Stratégie communautaire : chaque code-barres résolu (ou rempli à la main par un utilisateur) est mémorisé dans un index EAN. Aujourd'hui local (localStorage), demain partagé via Supabase → on construit progressivement notre propre base culturelle, indépendante des API externes. Plus il y a de testeurs, plus la base se remplit.
Journal des sessions
31 mai 2026
Session 7 — 🚀 Déploiement Cloudflare Pages (GitHub → auto-deploy) + script deploy.ps1 (copie, bump cache SW, commit, push en 1 commande). Onboarding email/SMS passé en interface inline (le prompt() ne marchait pas sur mobile). Mot de passe beta sur les versions testeurs. Scan caméra réparé (cycle de vie de l'élément, formats EAN forcés, haute résolution 1920×1080, autofocus continu, validation par double-lecture anti-faux-positif). Bases de données scan câblées : livres/mangas (Google Books + Open Library), CD/vinyles (Discogs + MusicBrainz), films/séries (OMDB), jeux de société (BGG). Constat : pas de base gratuite par code-barres pour jeux/DVD → stratégie d'index EAN local, futur socle d'une BDD culturelle COMMUNAUTAIRE (Supabase). 🔄 Tracker restructuré : priorités classées par besoin pour tester (inscription → sécurité → social → mobile → bases). Nouvelle idée : déconseiller une œuvre à un ami. Tableau des bases de données par type de support ajouté.
1 mai 2026
Session 6 — 🚀 Passage en PWA (Progressive Web App). Création du dossier dig_pwa avec manifest.json, service-worker.js (cache offline + stratégies réseau), icônes 192/512 générées avec branding teal/rose. App installable sur iPhone/Android sans App Store. Hébergement Netlify Drop. Onboarding renforcé avec vérification email + SMS (anti-bot). Mode nuit avec gradient dramatique teal/rose. Filtre statut en dropdown. Cards communauté harmonisées et compactées. Plan défini : PWA → Backend Supabase → Capacitor pour stores natifs.
